# 📘 README - Introdução ao Git e GitHub

## 🧠 O que é Git?

**Git** é um sistema de controle de versão distribuído, criado por Linus Torvalds em 2005. Ele permite que desenvolvedores acompanhem mudanças no código-fonte ao longo do tempo, colaborem em projetos e revertam alterações quando necessário.

### Principais comandos do Git:
```bash
git init           # Inicializa um repositório Git
git clone <url>    # Clona um repositório existente
git add <arquivo>  # Adiciona arquivos ao staging
git commit -m "mensagem"  # Salva alterações com uma mensagem
git push           # Envia alterações para o repositório remoto
git pull           # Atualiza o repositório local com mudanças remotas
```

## 🌐 O que é GitHub?

**GitHub** é uma plataforma baseada na web que hospeda repositórios Git. Ela facilita a colaboração entre desenvolvedores, oferece ferramentas de gerenciamento de projetos e integrações com CI/CD, entre outras funcionalidades.

### Recursos do GitHub:
- **Repositórios públicos e privados**
- **Issues** para rastrear tarefas e bugs
- **Pull Requests** para revisão de código
- **Actions** para automação de workflows
- **Wikis** e **GitHub Pages** para documentação e sites

## 🚀 Começando um projeto

1. Crie um repositório no GitHub.
2. Clone o repositório localmente:
   ```bash
   git clone https://github.com/seu-usuario/seu-repositorio.git
   ```
3. Faça alterações no código.
4. Use `git add` e `git commit` para salvar as mudanças.
5. Envie para o GitHub com `git push`.
